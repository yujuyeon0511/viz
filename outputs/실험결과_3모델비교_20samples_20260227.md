## 3. Information Flow 분석 (LLM Hidden States)

### 3.1 Inter-layer Cosine Similarity

이미지 토큰의 인접 레이어 간 코사인 유사도 변화:

| 모델 | 초기 레이어 | 중간 레이어 | 후기 레이어 | Cliff 위치 |
|------|------------|------------|------------|-----------|
| InternVL3.5-8B | ~0.85 | ~0.90-0.95 | 급락 ~0.60 | L30-33 (36L 중) |
| Qwen2.5-VL-7B | ~0.90 | ~0.93-0.96 | 급락 ~0.65 | L23-26 (28L 중) |
| LLaVA-SP | ~0.80 | ~0.85-0.90 | 완만한 하강 | L25-28 (32L 중) |

**핵심 발견**:
- InternVL과 Qwen은 후기 레이어에서 **급격한 cliff** 현상 관찰 → 이미지 토큰의 정보가 텍스트 토큰으로 이전되는 시점
- LLaVA-SP는 cliff가 상대적으로 완만 → 정보 전달이 덜 효율적
- Qwen은 28L로 가장 적은 레이어 수에도 cliff가 비례적으로 유사한 위치(~82%)에서 발생

### 3.2 L2 Norm 변화

| 모델 | 이미지 토큰 L2 Norm 추이 | 텍스트 토큰 대비 |
|------|-------------------------|-----------------|
| InternVL3.5-8B | 초기 낮음 → 후기 급증 | 후기에 텍스트 토큰과 유사 수준 |
| Qwen2.5-VL-7B | 초기 중간 → 후기 급증 | 후기에 텍스트 토큰과 유사 수준 |
| LLaVA-SP | 전반적으로 낮음 | 텍스트 토큰보다 consistently 낮음 |

### 3.3 Token Redundancy (Pairwise Cosine Similarity)

이미지 토큰 간 중복도 변화:

| 모델 | Image Token 수 | 초기 Redundancy | 최종 Redundancy | 증가 폭 |
|------|---------------|----------------|----------------|---------|
| InternVL3.5-8B | 256 (고정) | ~0.50 | ~0.85-0.95 | +0.35~0.45 |
| Qwen2.5-VL-7B | 84~840 (동적) | ~0.55 | ~0.88-0.96 | +0.33~0.41 |
| LLaVA-SP | 582 (고정) | ~0.45 | ~0.80-0.88 | +0.35~0.43 |

**핵심 발견**:
- 모든 모델에서 후기 레이어로 갈수록 이미지 토큰 redundancy 급증
- Qwen2.5-VL의 동적 토큰 수 (Spatial Merge 2×2): 이미지 크기에 따라 84~840개로 변동하나, redundancy 패턴은 일관
- InternVL은 Pixel Shuffle로 256개로 압축 → 초기부터 적절한 정보 밀도
- LLaVA-SP는 576+6=582개로 가장 많지만, 정보 밀도가 낮아 redundancy가 상대적으로 낮게 시작

## 4. ViT Attention 분석

### 4.1 방법론

| 모델 | ViT Attention 방식 | CLS Token |
|------|-------------------|-----------|
| InternVL3.5-8B | CLS→Patch attention | 있음 |
| Qwen2.5-VL-7B | Mean Attention Received (CLS 없음) | 없음 |
| LLaVA-SP | CLS→Patch attention | 있음 |

### 4.2 특성

- **InternVL**: CLS 토큰이 전체 이미지를 요약, 초기 레이어에서 균일 분포 → 후기에 특정 영역 집중
- **Qwen2.5-VL**: CLS 토큰 없이 Window+Full attention 혼합 → Full attention 레이어(L7,15,23,31)에서 전역 정보 통합
- **LLaVA-SP**: CLIP의 CLS→Patch attention → 범용 특징에 집중, 차트 특화 영역 약함

## 5. 이미지 토큰 효율성 분석

| 모델 | Image Tokens (평균) | 정확도 | 토큰당 정확도 | LLM Layers |
|------|-------------------|--------|-------------|------------|
| InternVL3.5-8B | 256 | 75.0% | 0.293%/token | 36 |
| Qwen2.5-VL-7B | ~395 (동적) | 70.0% | 0.177%/token | 28 |
| LLaVA-SP | 582 | 20.0% | 0.034%/token | 32 |

**Qwen2.5-VL 동적 토큰 분포 (20 samples)**:
- 최소: 84 tokens (S14: 15948.png, 작은 파이 차트)
- 최대: 840 tokens (S5: 3960.png, 큰 테이블 이미지)
- 평균: ~395 tokens
- 중간값: ~558 tokens

**핵심 발견**:
- InternVL이 가장 적은 토큰(256)으로 가장 높은 정확도 달성 → Pixel Shuffle의 효율성 입증
- Qwen2.5-VL의 동적 해상도는 이미지 복잡도에 맞게 토큰 수를 조절하지만, 평균 토큰 수는 InternVL보다 많음
- LLaVA-SP는 582개 토큰을 사용하면서도 가장 낮은 정확도 → 압축 없이 많은 토큰을 쓰는 것이 효율적이지 않음

## 6. 종합 결론

### 6.1 정확도 관점
1. **InternVL3.5-8B (75%)**: 20 samples에서 가장 높은 정확도. 수치 읽기, 카운팅, 비교 질문에 강함
2. **Qwen2.5-VL-7B (70%)**: InternVL과 근소한 차이. 동적 해상도로 다양한 이미지 크기에 적응
3. **LLaVA-SP (20%)**: ChartQA 특화 성능 부족. 단순 확인형(Yes/No) 질문에만 정답

### 6.2 Information Flow 관점
- **Cliff 현상**: InternVL과 Qwen 모두 LLM 후기 레이어(~80% 지점)에서 명확한 information cliff 관찰
- **Redundancy 패턴**: 3모델 모두 후기 레이어에서 redundancy 급증 → "From Redundancy to Relevance" 논문의 관찰과 일치
- **LLaVA-SP의 낮은 효율**: 높은 토큰 수에도 불구하고 information flow가 비효율적 → LoRA fine-tuning의 한계 가능성

### 6.3 토큰 압축 전략 비교
| 전략 | 모델 | 압축률 | 효과 |
|------|------|--------|------|
| Pixel Shuffle 0.5× | InternVL | 4:1 (1024→256) | 가장 효율적: 최소 토큰으로 최고 정확도 |
| Spatial Merge 2×2 | Qwen2.5-VL | 4:1 (동적) | 효율적: 이미지 적응형 토큰 수 |
| 압축 없음 + Spatial Conv | LLaVA-SP | 1:1 (576+6=582) | 비효율적: 많은 토큰으로 낮은 정확도 |

### 6.4 이전 실험 (3 samples) 대비 변화
| 모델 | 3 samples | 20 samples | 변화 |
|------|-----------|------------|------|
| InternVL3.5-8B | 3/3 (100%) | 15/20 (75%) | -25%p |
| Qwen2.5-VL-7B | 2/3 (67%) | 14/20 (70%) | +3%p |
| LLaVA-SP | 0/3 (0%) | 4/20 (20%) | +20%p |

→ 샘플 수 증가 시 InternVL의 정확도가 하락하고, LLaVA-SP는 소폭 개선됨. 20 samples가 더 신뢰할 수 있는 추정치.

## 8. 한계 및 향후 과제

1. **샘플 수**: 20개는 통계적 유의성에 한계. ChartQA 전체 1250개에 대한 실험 필요
2. **평가 기준**: 현재 수동 판정 → ChartQA 공식 relaxed accuracy (5% tolerance) 적용 필요
3. **Cliff-layer token truncation**: Information cliff 이후 레이어에서 이미지 토큰 제거 시 성능 변화 실험
4. **LLaVA-SP cropping variant**: pooling 대신 cropping 전략의 성능 비교
5. **더 다양한 차트 유형**: bar/pie/line 외에 table, scatter, heatmap 등에 대한 분석
